# Единственный источник истины: Backend + PostgreSQL

Все бизнес-данные приложения хранятся **только** в PostgreSQL и доступны **только** через Backend API (FastAPI). Фронтенд не пишет и не читает их из localStorage (кроме перечисленных ниже исключений).

## Что хранится в Backend (БД)

- Пользователи, задачи, проекты, таблицы, статусы, приоритеты
- CRM: клиенты, сделки, контракты, сотрудники, воронки
- Финансы: категории, фонды, план, заявки, документы, выписки, справки о доходах
- Контент: документы, посты, папки
- BPM: позиции, процессы; инвентарь: склады, товары, движения
- Настройки уведомлений (notification_prefs)
- Автоматизация (automation rules), активность (activity log)

Все эти данные загружаются при старте/при входе в модуль через `api.*` (см. `services/apiClient.ts`, `backend/api.ts`). Сохранение — только через вызовы API (PUT/POST/PATCH).

## Исключения (только в браузере)

В localStorage хранятся **только**:

- **Сессия:** `activeUserId` — кто сейчас «в системе» (до появления JWT можно не трогать).
- **UI:** тема (светлая/тёмная), свёрнута ли боковая панель, «не показывать день рождения сегодня».
- **Интеграции (опционально):** токены Telegram-ботов, chat ID — пока только в localStorage; при необходимости позже можно перенести в Backend (настройки в БД или env на сервере).

Файлы (аватар, вложения задач, сайты) пока могут храниться как data URL в localStorage; в перспективе — загрузка файлов через Backend.

## Что не используется

- **seed/mockData.ts** — не вызывается при старте приложения. Демо-данные при необходимости создаются через Backend (например, `server/seed.py` при первом запуске на сервере).
- **localStoreService** — использовался только для моков; данные приложения из него не читаются.
- **storageService** — методы чтения/записи бизнес-данных помечены как deprecated и не используются для источника истины; используются только сессия и (опционально) токены/настройки интеграций выше.

## Проверка

1. Запустить Backend (`npm run dev:server`) и Frontend (`npm run dev`).
2. Все списки (пользователи, задачи, финансы и т.д.) должны загружаться с API и сохраняться через API.
3. После перезагрузки страницы данные должны совпадать с тем, что в БД (нет «только локального» сохранения без записи в БД).
