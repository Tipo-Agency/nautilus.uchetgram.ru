# Техническое задание: Справка о доходах (Income Report)

## 1. Назначение

Функционал «Справка о доходах» входит в модуль финансового планирования и решает задачи:

- учёт поступлений по выпискам из банка (загрузка Excel);
- учёт данных о продажах/поступлениях из 1С;
- сверка банковских поступлений с данными 1С;
- формирование справки о доходах за выбранный период;
- автоматическая передача итогового дохода в финансовое планирование.

## 2. Общая логика

1. **Выписки из банка** — пользователь загружает Excel-выписку; система парсит файл и сохраняет операции в БД. Выписок может быть несколько (разные счета/банки).
2. **Данные из 1С** — подтягиваются поступления/продажи из 1С и сохраняются в БД.
3. **Сверка** — сопоставление сумм по банку и по 1С за период (расхождения подсвечиваются).
4. **Справка о доходах** — формируется за выбранный период. При формировании обязательно: либо загружена последняя выписка (или несколько), либо сумма дохода введена вручную. После создания справки сумма дохода автоматически подставляется в финансовое планирование за этот период.

---

## 3. Выписки из банка (Bank Statements)

### 3.1. Загрузка и парсинг

- **Формат входа:** Excel (`.xlsx` / `.xls`).
- **Действия:** загрузка файла → парсинг строк (дата, сумма, назначение платежа, контрагент и т.п. по формату банка) → сохранение записей в БД.
- Поддержка нескольких выписок: разные счета, разные банки — каждая выписка хранится отдельно с привязкой к счёту/банку.

### 3.2. Модель данных

**BankStatement (выписка — заголовок):**

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| name | string | Название (например, «Выписка Счёт XXX за 01.2026») |
| bankName | string | Название банка (опционально) |
| accountNumber | string | Номер счёта (опционально) |
| periodFrom | string | Начало периода (YYYY-MM-DD) |
| periodTo | string | Конец периода (YYYY-MM-DD) |
| uploadedAt | string | Дата/время загрузки (ISO) |
| uploadedByUserId | string | ID пользователя, загрузившего файл |
| totalIncome | number | Сумма поступлений по выписке (итог) |
| totalOutcome | number | Сумма списаний (опционально) |
| isArchived | boolean | Архив |

**BankStatementLine (строка выписки — одна операция):**

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| statementId | string | ID выписки (BankStatement) |
| date | string | Дата операции (YYYY-MM-DD) |
| amount | number | Сумма (положительная — приход, отрицательная — расход) |
| description | string | Назначение платежа / описание |
| counterparty | string | Контрагент (опционально) |
| documentNumber | string | Номер документа (опционально) |
| type | string | Тип: 'income' \| 'outcome' (или по знаку amount) |

### 3.3. API / хранилище

- Сохранение выписки и строк: `POST /api/finance/bank-statements` (или аналог в текущем API).
- Список выписок: `GET /api/finance/bank-statements` (с фильтром по периоду, счёту).
- Удаление/архив: `DELETE` или флаг `isArchived`.

Парсинг Excel — на бэкенде или отдельном сервисе; формат колонок можно задать в настройках (маппинг «колонка Excel → поле модели»).

## 4. Данные из 1С (Sales / Receipts)

### 4.1. Подтягивание данных

- Источник: 1С (API или выгрузка). Данные о продажах/поступлениях за период.
- Сохранение в БД в едином формате, привязанном к периоду и (при необходимости) к подразделению.

### 4.2. Модель данных

**IncomeFrom1C (поступления из 1С):**

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| periodFrom | string | Начало периода (YYYY-MM-DD) |
| periodTo | string | Конец периода (YYYY-MM-DD) |
| date | string | Дата операции (YYYY-MM-DD) |
| amount | number | Сумма поступления |
| description | string | Описание / документ 1С |
| documentRef | string | Ссылка на документ в 1С (опционально) |
| departmentId | string | ID подразделения (опционально) |
| syncedAt | string | Дата синхронизации (ISO) |
| isArchived | boolean | Архив |

Список и создание/обновление: `GET/POST /api/finance/income-1c` (или в рамках существующего finance API).

## 5. Сверка банк vs 1С

### 5.1. Назначение

- Показать за выбранный период:
  - сумму поступлений по выпискам (по банку);
  - сумму поступлений по 1С;
  - разницу (расхождение).
- Поддержка нескольких выписок: итог по банку = сумма итогов по всем выбранным выпискам за период (или сумма по строкам выписок с датой в периоде).

### 5.2. Экраны/блоки

- Выбор периода (дата с — дата по).
- Выбор выписок из банка за этот период (или автоматический подбор по датам).
- Таблица/карточки: период, сумма по банку, сумма по 1С, разница, статус (совпало / расхождение).
- Детализация: при клике — список операций по банку и по 1С с возможностью ручной сверки (опционально в ТЗ).

## 6. Справка о доходах (Income Report)

### 6.1. Суть документа

- **Справка о доходах** — документ за период (например, месяц), фиксирующий итоговую сумму дохода (кассовый метод). Эта сумма затем используется в финансовом планировании как фактический доход за период.

### 6.2. Правила формирования

- Пользователь нажимает кнопку **«Справка о доходах (за период)»** и выбирает период (например, месяц YYYY-MM или произвольные даты).
- **Обязательное условие перед формированием:**
  - либо загружена **последняя выписка** за этот период (или несколько выписок — разные счета/банки), и система считает итог по ним;
  - либо выписки нет — пользователь **вводит сумму дохода вручную**.
- Выписок может быть несколько; итог по справке = сумма по всем привязанным выпискам за период + при необходимости ручная корректировка (или только ручной ввод, если выписок нет).

### 6.3. Модель данных

**IncomeReport (справка о доходах):**

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| period | string | Период (YYYY-MM для месяца или YYYY-MM-DD—YYYY-MM-DD) |
| periodFrom | string | Начало периода (YYYY-MM-DD) |
| periodTo | string | Конец периода (YYYY-MM-DD) |
| amount | number | Сумма дохода (итог) |
| source | string | Источник: 'bank_statements' \| 'manual' \| 'mixed' |
| statementIds | string[] | ID выписок, по которым посчитан доход (если есть) |
| manualAmount | number | Введённая вручную сумма (если источник manual или mixed) |
| note | string | Комментарий (опционально) |
| createdAt | string | Дата создания (ISO) |
| createdByUserId | string | Кто сформировал |
| isArchived | boolean | Архив |

### 6.4. Сценарии

1. **Есть выписки за период:** пользователь выбирает период, система предлагает выписки за этот период; пользователь подтверждает (или выбирает подмножество) → система считает сумму поступлений по выбранным выпискам → создаётся справка с `source: 'bank_statements'`, `amount` = сумма по выпискам, `statementIds` заполнен.
2. **Выписок нет:** пользователь вводит сумму вручную → создаётся справка с `source: 'manual'`, `amount` = введённое значение, `manualAmount` = то же.
3. **Смешанный вариант:** часть по выпискам, часть вручную (например, один счёт по выписке, второй — ручной ввод) → `source: 'mixed'`, `amount` = сумма из выписок + ручная, `statementIds` и `manualAmount` заполнены.

### 6.5. Кнопка «Сформировать справку»

- При нажатии открывается экран/модалка:
  - выбор периода;
  - если есть выписки за период — список, галочки выбора, итоговая сумма;
  - поле «Сумма вручную» (если выписок нет или нужна доп. сумма);
  - итоговое поле «Доход за период» (только чтение или редактируемое при ручном вводе);
  - кнопка «Сформировать». После сохранения справки — автоматическая подтяжка дохода в финансовое планирование (см. п. 7).

---

## 7. Интеграция с финансовым планированием

- После успешного формирования **справки о доходах** за период:
  - система находит **финансовое планирование** (FinancialPlanning) за тот же период (и при необходимости подразделение, если планирование ведётся по подразделениям);
  - подставляет в поле **income** (доход за период) значение из справки (`IncomeReport.amount`);
  - при отсутствии записи финансового планирования за этот период — можно автоматически создать черновик с заполненным доходом или показать уведомление «Создайте планирование за период YYYY-MM и привяжите справку» (на усмотрение реализации).
- Связь «справка → планирование» можно хранить в FinancialPlanning, например: `incomeReportId?: string`, чтобы было видно, откуда взят доход.

## 8. Структура UI в модуле финансов

Рекомендуемая структура вкладок/разделов:

1. **Планирование** — текущий раздел финансового планирования.
2. **Заявки** — заявки на приобретение.
3. **План** — документы финансового плана (FinancialPlanDocument).
4. **Справка о доходах** (новый раздел):
   - подраздел **Выписки** — загрузка Excel, список выписок, просмотр строк;
   - подраздел **Данные 1С** — настройка синхронизации, список загруженных поступлений;
   - подраздел **Сверка** — выбор периода, сравнение банк vs 1С;
   - кнопка **«Сформировать справку за период»** — модалка/страница по п. 6.5;
   - список **сформированных справок** с периодом и суммой.

Либо один экран «Справка о доходах» с табами: Выписки | 1С | Сверка | Сформированные справки.

## 9. API (сводка)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | /api/finance/bank-statements | Список выписок (фильтр по периоду) |
| POST | /api/finance/bank-statements | Загрузка файла + парсинг, создание выписки и строк |
| GET | /api/finance/bank-statements/:id/lines | Строки выписки |
| GET | /api/finance/income-1c | Список поступлений из 1С (фильтр по периоду) |

| GET | /api/finance/income-reports | Список справок о доходах |
| POST | /api/finance/income-reports | Создание справки (период, выписки или ручная сумма) |
| GET | /api/finance/reconciliation | Данные для сверки за период (банк + 1С + разница) |

Формат и имена эндпоинтов согласовать с текущим бэкендом (REST / Firebase и т.д.).

## 10. Этапы реализации (рекомендуемый порядок)

1. **Модели и хранение:** BankStatement, BankStatementLine, IncomeFrom1C, IncomeReport в types и в БД/API.
2. **Загрузка выписок:** парсер Excel (формат колонок — конфигурируемый или фиксированный под один банк), сохранение выписки и строк.
3. **Интеграция 1С:** получение данных о поступлениях, сохранение в IncomeFrom1C.
4. **Сверка:** экран/блок с периодом, расчёт сумм по банку и 1С, отображение разницы.
5. **Справка о доходах:** UI выбора периода, выбора выписок / ручного ввода, создание IncomeReport.
6. **Автоподстановка в планирование:** при создании/обновлении справки — обновление income в FinancialPlanning за тот же период.

После реализации ТЗ справка о доходах будет давать единую точку ввода фактического дохода за период и автоматически подтягивать его в финансовое планирование.
